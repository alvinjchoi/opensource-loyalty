Everything between the order and the points

One protocol for the full loyalty lifecycle, designed for real restaurant operations.

🔄

Complete lifecycle

Evaluate orders before checkout, post accrual after payment, and run reserve, capture, reverse, and refund-safe adjustment flows.

🍔

Foodservice-first

Items, modifiers, discounts, fees, taxes, tips, and tenders, with channel rules for counter, drive-thru, kiosk, delivery, and catering.

🛡

Retry-safe by design

Idempotency keys with conflict detection, RFC 9457 problem details, and exact integer money. No double-posted points, no float drift.

🧩

TypeScript SDK

Typed client with automatic protocol context, exact-money helpers, a restaurant order builder, and webhook verification.

🖥

Local Admin dashboard

Inspect members, balances, tiers, expiring points, ledger history, and program configuration in a seeded local sandbox.

✅

Spec and conformance

OpenAPI 3.1 contract, JSON Schema payloads, and black-box conformance tests you can run against any implementation.

Integrate in an afternoon

The SDK supplies protocol version, request ids, timestamps, and idempotency keys on every call, and validates payloads before network I/O.

The full lifecycle example — enroll, evaluate, accrue, reserve, capture, reverse, refund — is fewer than 50 lines.

import { LipClient } from "@loyalty-interchange/sdk";

const lip = new LipClient({
  baseUrl: "http://127.0.0.1:3210",
  apiKey: process.env.LIP_API_KEY,
  source: { system: "ordering", instance: "prod" }
});

// Estimate points and rewards before checkout
const quote = await lip.orders.evaluate({
  member_id: memberId,
  order
});

// Post the earn after payment settles
await lip.accruals.post(
  { member_id: memberId, order },
  { idempotencyKey: `accrual:${order.order_id}` }
);
